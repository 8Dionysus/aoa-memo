# AoA Memo Charter

## Purpose

`aoa-memo` is the memory and recall layer of the AoA ecosystem.

Its purpose is to make memory explicit, bounded, and reviewable rather than leaving it as an invisible side effect of context windows or scattered project notes.

## Mission

This repository exists to support memory-oriented work such as:
- storing memory objects with clear shape
- preserving provenance and recall pathways
- separating episodic and semantic memory surfaces where useful
- making temporal relevance and salience more legible
- supporting agents and humans in recalling prior work without confusing recall with proof

## What this repository owns

This repository owns memory-layer truth about:
- memory objects
- recall surfaces
- provenance threads
- temporal and salience-oriented memory surfaces
- memory-layer schemas and validation rules
- memory boundaries relative to practice, execution, proof, and routing

## What this repository does not own

This repository does not own the primary meaning of:
- reusable techniques
- agent workflows
- bounded proof surfaces
- dispatch and routing
- infrastructure implementation

## Core principles

- memory should be explicit rather than magical
- provenance should remain legible where possible
- recall must stay bounded enough to review
- memory should support reuse without silently rewriting source truth
- temporal and salience surfaces should be treated as contracts, not vibes
- memory is not proof and should never quietly replace proof

## Source-of-truth rule

Neighboring AoA repositories still own their own meaning.

Examples:
- `aoa-techniques` owns practice meaning
- `aoa-skills` owns execution meaning
- `aoa-evals` owns proof meaning
- `aoa-routing` owns navigation and dispatch surfaces
- `aoa-memo` owns memory and recall meaning
- `aoa-agents` should eventually own role and persona meaning

## Role in the federation

Treat `aoa-memo` as:
- the memory-layer repository of AoA
- the canonical home of memory and recall contracts
- the place where memory objects and registry surfaces become reviewable

## Long-term direction

If `aoa-memo` matures well, it should help AoA accumulate memory without collapsing the distinction between:
- what happened
- what is reusable
- what should be done
- what can be honestly proven
