# VIA_NEGATIVA_CHECKLIST

This checklist is for `aoa-memo` as the reviewable memory and recall.

## Keep intact

- reviewed memory objects with explicit trust posture
- evidence refs, recall prompts, and supersede paths
- object families that clearly differ in recall purpose

## Merge, move, suppress, quarantine, deprecate, or remove when found

- memory objects that imply live health or truth
- object kinds that differ only by a couple of field names
- scalar confidence fields that collapse split trust posture into one number

## Questions before adding anything new

1. Is this memory or proof?
2. Would a reviewed recall object work instead of a new object family?
3. Does this object trigger action without owner receipts or eval backing?

## Safe exceptions

- a genuinely new recall family with distinct retrieval semantics
- temporary bridges for migrating consumers away from a bad shape

## Exit condition

- Memo should remember clearly without pretending to certify reality.
