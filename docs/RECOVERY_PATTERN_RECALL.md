# RECOVERY PATTERN RECALL

## Goal

Recall reviewed recovery patterns when they sharpen posture across repeated
stress windows, not when they merely decorate the route with extra narrative.

## Preferred query shapes

Useful queries are usually narrow:

- stressor family + repeated window
- repo family + bounded recovery posture
- eval report + stats summary + route hint
- degraded route + re-entry readiness

Examples:

- `hybrid-query-kag-unhealthy + stress recovery window`
- `retrieval_only_fallback + repeated bounded recovery`
- `route hint + reviewed recovery pattern`

## What a recall result should answer

A bounded recall result should help answer:

- have we seen this recovery posture survive an ordered window before
- which stronger sources should we inspect first
- what remained bounded during recovery
- what should remain blocked until stronger evidence is checked

## Precedence reminder

For current truth, prefer:

1. source-owned receipts and handoff artifacts
2. bounded eval reports
3. derived stats summaries and reviewed route hints
4. memo recovery-pattern recall as contextual support

Memo may shape recall and routing review.
It does not overrule source-owned receipts, eval proof, or derived stats
summaries.

## Freshness and suppression

Suppress or cool recall when:

- the linked window is stale
- the pattern is still a draft
- the linked stats summary reports thin evidence suppression
- a newer reviewed pattern supersedes it

## Suggested recall surface behavior

A healthy recovery-pattern recall surface should expose:

- the recovery summary
- the ordered window description
- linked stronger evidence refs
- trust posture
- one bounded posture sentence
- one note about what remains uncertain
