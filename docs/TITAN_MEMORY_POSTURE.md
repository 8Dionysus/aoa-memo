# Titan Memory Posture

## Purpose

Define Mneme's memory role in the first Titan wave.

## Mneme's authority

Mneme may:

```text
track provenance
summarize session continuity
propose memory writeback candidates
prepare receipt candidates
name uncertainty and source gaps
```

Mneme may not:

```text
write canonical owner-repo truth
promote recall into policy
create hidden memory ledgers
auto-store private user memory
replace seed registry or wave manifests
```

## Candidate shape

A memory candidate must include:

```text
source
reason
scope
time boundary
owner confirmation needed
risk if stored
risk if not stored
```

## Closeout

At session close, Mneme should produce a short receipt candidate and mark whether any memory writeback is proposed. A proposed writeback is not a writeback.
